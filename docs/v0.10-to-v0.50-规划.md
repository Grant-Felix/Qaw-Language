# 从 v0.10 到 v0.50 规划

> 本文档是 Qaw 从 **v0.10**（当前）到 **v0.50 = 完整独立的开发语言** 的规划。
> 文档基于主流编程语言（C/C++/Rust/汇编/Java/Go/C#/Python/Scala/Ruby/JS/TS/Swift/Kotlin/Dart/R/Lua/GDScript）的特性研究。
>
> **状态**：规划草案（2026-06-30），随实施进展调整。

---

## 1. 规划原则

1. **每个小版本 = 一个最小不可分割独立改变**（已在 21-路线图 §21.1 定义）
2. **任务来源**：仅从本研究确认的主流语言特性池中选取
3. **不做的事**：明确写出，避免范围蔓延
4. **依赖排序**：底层特性先于高层特性（如类型系统先于标准库）

### 已实现

| 类别 | 已有 | 来源 |
|:---|:---|:---|
| 词法 | UTF-8、四形制关键字、字符串字面量、注释 | v0.1 |
| 语法 | if/while/for-range/match、函数、struct/enum | v0.1 |
| 求值 | Tree-walking 解释器、内置 print、结构化 EvalError | v0.1, v0.5 |
| 模块 | 模块化源码组织（parser/、ast/ 子目录） | v0.2, v0.3 |
| 工具链 | qawc CLI（lex/parse/run/build）、clippy 6 warnings | v0.1, v0.4 |
| CI | GitHub Actions（Linux、cargo + clippy + 端到端） | v0.6 |
| 文档 | 蓝图 23 章 + 4 附录（约 5900 行）、6 条设计原则 | v0.0, v0.8, v0.9 |
| 治理 | 路线图重构为任务清单驱动 + 分支治理章节 | v0.7, v0.10 |

### 缺口（vs 主流语言最小集）

- 无类型系统完整性（无泛型、无 trait、无可见性）
- 无内存安全（无借用检查、无 Drop 检查）
- 无标准库（无 Vec/HashMap/String/IO）
- 无包管理（无 qawpm）
- 无 FFI
- 无异步/并发
- 无真正编译产物（仅 fallback 到解释器）
- 无完整工具链（无 fmt / doc / test / LSP）

## 3. 语言特性池（从研究得出）

### 3.1 共性特性（≥80% 语言都有，Qaw v0.50 必须具备）

| 类别 | 特性 | 来源 |
|:---|:---|:---|
| 类型 | 基本类型（int/float/bool/string/char/array/struct/enum） | 所有 |
| 控制流 | if/else、for/while、match/case、函数 | 所有 |
| 函数 | 默认参数、关键字参数、可变参数、闭包 | Python/C++/Swift/JS |
| 错误 | 异常机制 + Result 风格 | Java/Python/Rust |
| 模块 | import / use / pub 可见性 | 所有 |
| 标准库 | 字符串、集合、IO、文件、JSON | 所有 |
| 构建 | 包管理（init/add/install/lockfile） | Cargo/pub/SPM/npm |
| 诊断 | 错误信息含位置、可读、有建议 | Rust/Elm |

### 3.2 优秀特性（择其精华）

按对 Qaw 目标（6 条原则）的契合度排序：

| 优先级 | 特性 | 来源语言 | Qaw 应用 |
|:---:|:---|:---|:---|
| 🏆 P0 | **Sound null safety**（`T?`/`T` 在类型系统层区分） | Dart | 替代当前 `Value::Nil`，编译期杜绝空指针 |
| 🏆 P0 | **值类型一等公民**（struct/enum 默认栈分配） | Swift | Qaw 默认高性能，无 GC 压力 |
| 🏆 P0 | **协议/接口扩展**（给已有类型加方法） | Swift/Rust trait | 标准库扩展机制 |
| 🏆 P0 | **Result<T,E> + ? 运算符** | Rust | 替代当前 EvalError 字符串化 |
| 🏆 P0 | **Block/yield 控制流抽象** | Ruby | with/transaction/iterator 统一 |
| 🥇 P1 | **模式匹配（type/list/or）** | Rust/Python PEP 634 | match 表达式扩展 |
| 🥇 P1 | **defer 延迟清理** | Go/Swift/C++ RAII | 资源管理 |
| 🥇 P1 | **Iterator + lazy combinators** | Rust/Java Streams/C# LINQ | 标准库集合操作 |
| 🥇 P1 | **LINQ 风格统一查询** | C# | 集合+异步统一 |
| 🥇 P1 | **位置/关键字专用参数分隔符（`/` `*`）** | Python | API 稳定性 |
| 🥇 P1 | **Checked exception + Result 二轨** | Java/Rust | 错误处理双通道 |
| 🥈 P2 | **Generics with Concepts 风格约束** | C++20/Rust trait | 泛型系统 |
| 🥈 P2 | **协程 + channel（CSP）** | Go | 并发基础 |
| 🥈 P2 | **协程 + actor** | Dart/Swift | 并发隔离模型 |
| 🥈 P2 | **整数溢出检查（默认开启）** | Swift | 默认安全 |
| 🥈 P2 | **async/await + Stream** | Dart/Kotlin | 异步 |
| 🥈 P2 | **生成器 + 推导式** | Python | 集合构造 |
| 🥈 P2 | **循环 else 子句** | Python | for-else 模式 |
| 🥈 P2 | **with/context manager** | Python | 资源作用域 |
| 🥉 P3 | **checked exception 单独强制** | Java | 不引入（见 4.5） |
| 🥉 P3 | **运行时反射（macro_rules 之外）** | Rust | 不引入 |

### 3.3 Qaw 明确不做的

| 特性 | 不做的原因 |
|:---|:---|
| Checked exception 单独强制 | Java 实践痛苦，与原则 4（默认安全但不繁琐）冲突 |
| 运行时反射（TypeId / Any::type_id） | 与原则 4（默认禁用不安全 + 低质量）冲突 |
| 元编程（macro 之外的 codegen） | 与原则 2（代码式）冲突，留给生态 |
| 模板元编程（SFINAE 类） | 复杂度高，C++20 Concepts 已替代 |
| 自省式 GC 调优接口 | 与原则 3（无运行时开销）冲突 |
| 动态类型默认 | 与原则 4 冲突 |

## 4. v0.50 任务清单（50 + 14 = 64 项）

按依赖排序，分 5 个阶段：

### 阶段 A：基础类型与安全（11 项，对应 v0.11-v0.21）

| # | 任务 | 来源 | 验收标准 |
|:---:|:---|:---|:---|
| A1 | **Sound null safety**（Dart 风格） | Dart | 编译器保证 `T` 不可能为 null；`T?` 显式可空；当前 `Value::Nil` 改造 |
| A2 | **值类型 vs 引用类型**（struct 默认栈） | Swift | `struct` 默认栈分配；`class` 显式堆 |
| A3 | **整数溢出检查（debug 模式）** ✅ 已完成 | Swift | 调试模式溢出 panic；release 模式 wrap — v0.12 实现（debug panic + release wrap） |
| A4 | **字符串基础操作**（len/concat/slice/contains） ✅ 已完成 | 所有 | UTF-8 安全；标准库暴露 — v0.13 实现 |
| A5 | **数组类型 `[T; N]` + slice `[T]`** | Rust | 固定/动态数组分离 |

每条原则由对应阶段任务落地：

| 原则 | 主要落地任务 |

## 8. 风险与备选

| 风险 | 影响 | 缓解 |
|:---|:---|:---|
| 借用检查器实现复杂度 | C 阶段延期 | 先实现单所有者 + 不可变借用（NLL 最小集），不变检查可推到 v0.100 |
| LLVM 集成延期 | D 阶段延期 | 先评估 Cranelift（Rust 生态 `cranelift-codegen`），比 LLVM 简单 50% |
| 泛型约束表达力不足 | D2 受限 | 先实现单态化（rustc 早期方案），trait object 后做 |
| 标准库 vs Qaw 语言版本同步 | 维护负担 | Qaw 语言锁版前标准库不锁版；标准库作为独立 crate |
| async 与 sync 互操作 | C 阶段复杂度 | Rust 异步的历史教训：先 sync 后 async |
| 整数溢出检查影响性能 | A3 | release 模式默认 wrap；用户可显式 `checked` 标记 |
