# RFC 模板

> 状态：草案 / 接受 / 拒绝 / 撤回 / 替换
> 作者：姓名 <邮箱>
> 创建日期：YYYY-MM-DD
> 最后更新：YYYY-MM-DD

## 摘要

一两句话说明本 RFC 提议什么。

## 动机

为什么需要这个？解决什么问题？痛点是什么？

## 设计目标

本 RFC 成功后应达到什么目标。

## 详细设计

具体的设计方案：语法、语义、API、影响范围。

## 权衡

列出所有备选方案，并说明为何选这个。

## 缺点

本设计的风险、副作用、迁移成本。

## 未来可能性

后续可能扩展的方向。

## 未决问题

任何需要进一步讨论的子问题。

## 实现计划

- [ ] 阶段 1：...
- [ ] 阶段 2：...
- [ ] 阶段 3：...

## 替代方案

为何不选其它方案。
