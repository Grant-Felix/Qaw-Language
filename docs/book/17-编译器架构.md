## 17.1 实现策略

- **当前阶段（v0.0 → v1.0）**：用 **C / C++** 实现编译器 `yaoc`（参考 GCC 思路）
- **远期目标**：妖语言成熟后，逐步用妖语言自身重写 `yaoc` 各模块
- 重写完成的模块从 C 版退役，保留作为参考实现

## 17.2 总体架构

```
源文件 (.yao)
    ↓
词法分析 (Lexer)
    ↓ Token 流
语法分析 (Parser)
    ↓ AST
AST 规范化
    ↓ HIR
名称解析 + 宏展开
    ↓ THIR
类型检查
    ↓ MIR
借用检查
    ↓ MIR
单态化
    ↓ LIR
优化 Pass
    ↓
代码生成 (LLVM IR / WASM)
    ↓
目标文件 / WASM / 二进制
```

## 17.3 前端

- **词法分析**：UTF-8 输入 → Token 流。
  - 四形制关键字归一化。
  - 字符串插值 `${}` 在词法层处理。
  - 嵌套块注释。
  - 零宽字符检测。
- **语法分析**：手写递归下降 + Pratt 解析运算符。
  - 增量解析支持。
  - 多错误聚合。

## 17.4 中端

- **HIR（High-level IR）**：AST 规范化。
- **THIR（Typed HIR）**：类型检查的中间表示。
- **MIR（Mid-level IR）**：CFG 形式，用于借用检查与优化。
- **借用检查**：NLL 算法。
- **模式穷尽性检查**。
- **Drop 检查**。
- **单态化**：每个泛型实例生成独立代码。

## 17.5 后端

| 后端 | 用途 | 实现 |
|:---|:---|:---|
| LLVM | 主后端 | `llvm-sys` / 直接 C API |
| WASM | 浏览器 | v2.0 实现 |

## 17.6 自举路径

**阶段 0：C / C++ 引导版（`yaoc-c`）**

- 实现最小子集（参考 22.15 节）。
- 能编译自身源码。

**阶段 1：妖语言自举过渡版（`yaoc-yao-v0`）**

- 用妖语言 v0 写。
- 由 C 引导版编译。

**阶段 2：完整自举（`yaoc-v1`）**

- 用妖语言 v1 写。
- 由自举过渡版编译。
- **字节级一致性验证**：编译自身源码，产物与 C 引导版编译同一源码的产物**必须字节级一致**。

**阶段 3：自举完成**

- 完全用自举版。
- C 引导版仅作参考。
