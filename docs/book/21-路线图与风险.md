## 21.2 详细路线图

### v0.0 — 蓝图阶段（第 0-6 月）

- ✅ 完成 23 章完整设计蓝图
- ⬜ 起草 5+ 个 RFC
- ⬜ 社区评审

### v0.1 — POC（第 6-9 月）

- ✅ Rust 引导版 `qawc`
- ✅ 词法分析器（支持四形制关键字）
- ✅ 语法分析器（基础语法 + 控制流）
- ✅ 简单解释执行（Tree-walking）
- ✅ Hello World 运行成功（输出"你好, 世界! 🌍"）

### v0.5 — MVP（第 9-15 月）

- ⬜ LLVM 后端集成
- ⬜ @safe 模式 + 借用检查
- ⬜ @dynamic 模式（基础）
- ⬜ 模块系统
- ⬜ 简单 qawpm + fmt

### v0.9 — Beta（第 15-21 月）

- ⬜ 泛型 + trait 完整
- ⬜ 协程 + 通道
- ⬜ 错误处理完整
- ⬜ FFI（C ABI）

### v1.0 LTS（第 21-27 月）

- ⬜ Python 互操作
- ⬜ 完整标准库
- ⬜ Rust 引导版 + 妖语言自举双编译验证
- ⬜ 包注册表
- ⬜ 三平台发布（Linux / macOS / Windows）

### v2.0（第 27-39 月）

- ⬜ WASM 后端
- ⬜ 完整异步运行时
- ⬜ @shared 模式（ARC）
- ⬜ 过程宏

## 21.4 技术风险

### 高风险

| 风险 | 影响 | 缓解 |
|:---|:---|:---|
| LLVM 后端集成 | 高 | 使用成熟绑定（llvm-sys / 自写 C 绑定） |
| 借用检查器复杂度 | 高 | 参考 Rust NLL；先实现简单版本 |
| GMP 调度器实现 | 高 | 参考 Go runtime；早期用单线程 |
| Rust 自举路径 | 高 | 提前准备引导版；预留 6 个月 |

### 中风险

| 风险 | 影响 | 缓解 |
|:---|:---|:---|
| 增量编译性能 | 中 | 早期用全量；后期优化 |
| WASM 后端成熟度 | 中 | v2.0 实现；早期用 LLVM 转译 |
| 跨平台兼容性 | 中 | tier 1 优先；tier 2/3 后续 |
| 生态冷启动 | 中 | 提供 FFI 到 Python/C；标准库丰富 |

### 低风险

| 风险 | 影响 | 缓解 |
|:---|:---|:---|
| CI 成本 | 低 | GitHub Actions / 自建 |
| 域名 / 基础设施 | 低 | 标准云服务 |

## 21.5 回退方案

| 风险 | 回退方案 |
|:---|:---|
| v1.0 无法按时 | 延期至 30 月，或先发 0.9 LTS |
| 自举失败 | 保留 Rust 引导版与 Qaw 版并存 |
| LLVM 切换 | 锁定 LLVM 主版本或换 Cranelift |
| 标准库困难 | 通过 FFI 依赖 C 库，或拆分为 std + std-contrib |

## 21.6 后续阶段（v3.0+）

| 阶段 | 目标 |
|:---|:---|
| v3.0（v2.0 之后约 1 年） | 异步统一（消除 goroutine / async 二分） |
| v3.5 | 编译时 AI 辅助 |
| v4.0（v3.0 之后约 1-2 年） | 类型系统扩展（HKT、dependent types 实验） |
| v5.0 | 真正的自举 + 自描述编译器 |

各阶段的具体时间根据实际进展调整。
