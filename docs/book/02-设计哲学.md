## 2.1 三统原则

### 书同文：四形等价

每个关键字同时支持四种书写形态：

| 形态 | 示例 | 适用 |
|:---|:---|:---|
| 英文原形 | `package`, `func`, `return` | 国际开发者 |
| 英文缩写 | `pkg`, `fn`, `ret` | 追求简洁 |
| 汉语全拼 | `baozhuang`, `hanshu`, `fanhui` | 中文母语 |
| 拼音首字母 | `bz`, `hs`, `fh` | 极致简洁 |

四种形态在词法层归一化为同一个 Token，源码中可任意混用。

### 车同轨：LLVM + WASM 双后端

同一份源码，跨平台原生产出。

| 后端 | 用途 |
|:---|:---|
| LLVM 17+ | 服务器、桌面、嵌入式 |
| WASM | 浏览器、边缘、插件沙箱 |

### 统一度量衡：三大内存模式

| 模式 | 标注 | 适用 |
|:---|:---|:---|
| 安全模式 | `@safe`（默认） | 90% 场景 |
| 共享模式 | `@shared` | 图、UI 树 |
| 动态模式 | `@dynamic` | 胶水、脚本 |

## 2.2 四大铁律

1. **取其精华，去其嫁接**：不发明新轮子。
2. **默认最聪明，特例最自然**：编译器能推导的，绝不让人写。
3. **不拒绝历史，不恐惧未来**：通过 C ABI / Python import 吃掉存量。
4. **极端向后兼容，平滑增量升级**：永不抛弃历史用户。

## 2.3 与主流语言对比

| 维度 | Rust | Go | Python | **妖语言** |
|:---|:---|:---|:---|:---|
| 内存安全 | 编译期 | GC | GC | 编译期（默认）/ ARC / 动态 |
| 开发效率 | 低 | 高 | 极高 | 高（自动推导） |
| 并发模型 | 线程 | GMP 协程 | GIL 受限 | GMP 协程 + Rust 通道安全 |
| 跨平台 | LLVM | 原生 | 解释器 | LLVM + WASM |
| 生态存量 | 重写 | 重写 | 最丰富 | 直接 import C / Python |
| 学习曲线 | 极陡峭 | 平缓 | 平缓 | 平缓（四形可选） |

## 2.4 不做的事

明确放弃的事，避免误以为有银弹：

1. **不挑战 LLVM 的代码生成质量**。我们用 LLVM。
2. **不挑战 V8 的 JS 引擎优化**。我们编译到 WASM，让浏览器原生引擎跑。
3. **不挑战 CPython 的科学栈**。我们直接 import。
4. **不挑战 Rust 的安全证明**。我们借用 Rust 的所有权模型，承认其正确性。
5. **不发明新范式**。所有范式都来自现有语言。

唯一的新东西，是"四形制关键字"这一语法皮层。
