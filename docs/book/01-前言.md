## 1.1 什么是妖语言

妖语言（Qaw Language）是一门通用编程语言。

它不发明新概念。每个关键字、每个语法特性都直接取自现有主流语言中受众最广的形式：

- 所有权与借用 → 来自 Rust
- 协程与通道 → 来自 Go
- 模式匹配 → 来自 Rust
- 推导式与上下文管理器 → 来自 Python
- 错误传播 `?` → 来自 Rust
- 宏 → 来自 Rust
- 注解 → 来自 Java / C#
- 字符串插值 → 来自 Kotlin / Swift
- 命名参数 → 来自 Swift / Python

妖语言做的是把这些特性统一到一套语法皮层下，用四形制关键字（英文 / 英文缩写 / 汉语全拼 / 拼音首字母）支持不同习惯的开发者。

## 1.2 与现有语言的关系

| 语言 | 妖语言取自它的特性 |
|:---|:---|
| C / C++ | FFI、嵌入汇编、`#repr(C)` 布局 |
| Rust | 所有权、生命周期、trait、宏 |
| Go | `go` 协程、`chan` 通道、`defer` |
| Python | `importPython`、列表推导式、`with` |
| Java / C# | `@attr(...)` 注解 |
| Kotlin / Swift | 字符串插值 `${expr}` |
| TypeScript | 可选链 `?.`、空值合并 `??` |

详细设计哲学见 [第 2 章](02-设计哲学.md)。

## 1.3 阅读对象

- 想用 Rust 写系统但觉得生命周期标注繁琐的人
- 想用 Go 写并发但受够了 GC 暂停的人
- 想用 Python 做数据但受不了 GIL 的人
- 想让孩子学一门编程语言入门的人

## 1.5 本书约定

- **示例代码**用 ` ```yao ` 代码块包裹。
- **输出**用 ` ```text ` 代码块。
- **命令**用 ` ```bash ` 代码块。
- **配置文件**用 ` ```toml ` 代码块。
- 关键字四形等价，例如 `func` 与 `hanshu` 完全等价。
- 所有跨章引用通过文件名，不依赖章节编号（章节编号可能因内容增减而变化）。
