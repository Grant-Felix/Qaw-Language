# 反馈：用户期望与旧语言吐槽

征集两类内容：

## 1. 对新语言的期望

你希望一门**新的编程语言**做到什么？

例如：
- "我希望有一种语言能同时像 Rust 一样安全，像 Python 一样简洁"
- "我希望类型系统能自动推导，不要写一堆 `let x: int`"
- "我希望字符串拼接不需要 `format!`，直接用 `${}`"
- "我希望 async/await 不要和同步代码割裂"

## 2. 对旧语言的吐槽

你在使用现有语言时，最受不了的是什么？

例如：
- "Rust 的生命周期标注太繁琐"
- "Go 的错误处理 `if err != nil` 污染业务逻辑"
- "Python 的 GIL 让我多线程没用"
- "JavaScript 的 `this` 指向混乱"
- "C++ 的模板错误信息像天书"

## 提交方式

请在 GitHub / Gitee Issues 中提交，标题以 `[反馈]` 开头。

## 用途

所有反馈将进入 [RFC 流程](docs/rfcs/)：
- 高频痛点 → 优先实现
- 期望但当前语言未做到的特性 → 设计参考
- 对现有语言的具体批评 → 借鉴与避免

每个季度汇总一次，纳入 [`docs/3-总任务表和进度.md`](docs/3-总任务表和进度.md)。
